Serveur Web Full‑Stack (Raspberry Pi)

Projet Personnel — Mise en place d’un serveur web complet (React/Vite + PHP API + MariaDB + Apache2).

← Retour aux projets
Résumé

Transformer un Raspberry Pi en serveur web capable d’héberger une application front React (Vite), une API PHP et une base MariaDB/MySQL, accessible via DuckDNS + HTTPS (Let’s Encrypt).

Stack
Matériel
  • Raspberry Pi (Pi 4 / Pi 5 recommandé)
  • Alimentation USB‑C
  • Stockage microSD 32–64 Go (ou SSD optionnel)
  • Boîtier ventilé + dissipateur/ventilateur
  • Réseau Ethernet (recommandé) / Wi‑Fi
Étapes / Visuels
Matériel
  • Raspberry Pi (Pi 4 / Pi 5)
  • Alimentation USB‑C
  • microSD 32–64 Go
  • Boîtier ventilé + dissipateur
  • Ethernet / Wi‑Fi
DuckDNS
  • DDNS gratuit (.duckdns.org)
  • Accès au serveur même si l’IP change
  • HTTPS (Let’s Encrypt) + redirection routeur
Installation / Accès
  • SSH : connexion distante + sécurisation (clé, ports, firewall)
  • PHP : installation + modules nécessaires + config Apache
  • MariaDB : installation, users, droits, dump/restore
  • VNC (bonus) : accès bureau à distance si besoin
  • Box/Routeur : ouvrir/rediriger les ports 80 (HTTP) et 443 (HTTPS) vers le Raspberry
