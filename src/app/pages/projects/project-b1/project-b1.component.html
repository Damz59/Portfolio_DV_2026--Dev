Base de données SDBM — Conception & implémentation
  • Conception d’une base complète à partir d’un existant phpMyAdmin (dump XAMPP/MariaDB).
  • Modélisation : articles/bières, marques, fabricants, couleurs, types, pays, continents + tickets/lignes.
  • Règles : recherche multi-critères, calcul prix (marge + TVA), unicité “1 bière = 1 ligne” par ticket.
  • Implémentation SQL Server : PK/FK, CASCADE/SET NULL/NO ACTION, contraintes UNIQUE/CHECK.
  • Vue de calcul HT/TTC + numérotation “sans trous” + tests (INSERT).
Back-end — Java + Spring Boot
  • Structure : Controller → Service → Repository → MariaDB
  • Config : application.properties (XAMPP/MariaDB, JPA validate)
  • API REST : endpoints stables + ResponseEntity
  • Recherche : Specifications + /api/article/search
  • Contrat : DTO + Mapper (Entity ↔ DTO)

GET /api/article/all • /id/{id} • /volumes • /search

Front-end — Angular (SPA)
  • Initialisation : Angular CLI (sans SSR) + routing + composants standalone.
  • Pages : Home + Read.
  • Composants métier : filtres + recherche par ID.
  • Service HTTP : appels API vers Spring Boot.
  • UI : Bootstrap + gestion loading/erreurs.

Dev : ng serve — App : localhost:4200
